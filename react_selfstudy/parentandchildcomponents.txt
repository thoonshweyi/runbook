Example: Parent and Child Wrapped Components
function App() {
  console.log("App render");
  return (
    <Parent>
      <Child />
    </Parent>
  );
}

function Parent({ children }) {
  console.log("Parent render");
  return (
    <div>
      <h2>Parent Component</h2>
      {children}
    </div>
  );
}

function Child() {
  console.log("Child render");
  return <h3>Child Component</h3>;
}

What happens when React renders this?
Step 1: App renders

React calls App()

Logs: "App render"

Returns <Parent> with <Child> inside

Step 2: Parent renders

React calls Parent({ children: <Child /> })

Logs: "Parent render"

Returns <div> â€¦ {children} </div>

Step 3: Child renders

React sees {children} â†’ calls Child()

Logs: "Child render"

Returns <h3>Child Component</h3>

âœ… Console Output (Render Order)
App render
Parent render
Child render

Key Points

Parent renders first â†’ React computes its JSX

Children render after â†’ React calls them in the order they appear

This is true even if children are wrapped inside a div or multiple layers

Step 5: State Updates Example
function Parent({ children }) {
  const [count, setCount] = React.useState(0);
  console.log("Parent render", count);

  return (
    <div>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      {children}
    </div>
  );
}


Click the button â†’ Parent re-renders â†’ Child also re-renders

React always re-renders children after parent
